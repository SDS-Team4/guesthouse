# IMPLEMENT

Codex는 이 문서를 실행 규칙(runbook)으로 사용한다.

## 운영 규칙

1. `PLANS.md`의 현재 milestone만 수행한다.
2. milestone 밖의 작업은 하지 않는다.
3. 구현 전에 관련 SRS 요구사항 ID를 다시 확인한다.
4. DB 변경이 있으면 스키마 문서와 영향 범위를 함께 갱신한다.
5. 검증 실패 시 다음 단계로 넘어가지 않는다.
6. 중요한 설계 판단은 `STATUS.md`에 남긴다.
7. open question이 남아 있으면 추측으로 구현하지 않는다.

## 응답 형식

작업 후 아래 순서로 요약한다.

1. 변경 파일
2. 관련 요구사항
3. 검증 결과
4. 남은 리스크
5. 다음 한 단계
