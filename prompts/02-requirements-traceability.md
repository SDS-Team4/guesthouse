Using the SRS and PLANS.md, create or refine a requirement traceability map.

Tasks:
1. Group requirements by capability:
   - auth/account recovery
   - guest discovery
   - reservation request and locking
   - guest reservation management
   - host inventory/pricing/block
   - host reservation ops
   - admin operations
   - nonfunctional/security
2. Mark each group as:
   - MVP
   - post-MVP
   - blocked by open question
3. For each group, list:
   - requirement IDs
   - required entities/tables
   - main APIs
   - main UI screens
   - validation approach
4. Suggest milestone boundaries that minimize cross-team coupling.

Do not implement code.
Write results into docs/plan/PLANS.md or a new docs/plan/TRACEABILITY.md if needed.
